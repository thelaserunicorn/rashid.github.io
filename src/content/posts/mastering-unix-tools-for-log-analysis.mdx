You've been there. It's 2 AM and your phone buzzes with an alert. Production is degrading. You SSH into the server, open a 4 GB access log file, and start scrolling... only to realize you've been staring at the same block for ten minutes.

Sound familiar? You're not alone. This is the reality of production debugging, and it's exactly why three ancient Unix tools—**grep**, **sed**, and **awk**—remain absolute superpowers in every senior engineer's toolkit.

These tools were built for exactly this scenario: extracting signal from noise, transforming raw data into actionable insights, and doing it fast enough to matter when your users are waiting.

Let's dive in.

## The Sample Data

Throughout this guide, we'll use a single realistic Nginx access log. Copy this into a file called `access.log`:

```bash
2026-05-23 02:14:32 INFO  192.168.1.105 - GET /api/v2/users 200 45ms
2026-05-23 02:14:33 INFO  10.0.3.87 -  GET /api/v2/products 200 38ms
2026-05-23 02:14:35 WARN  192.168.1.203 - POST /api/v2/orders 429 1203ms
2026-05-23 02:14:36 ERROR 10.0.2.15 -  GET /api/v2/inventory 500 892ms
```

## Section 1: grep — The Scout

**When to use it:** You need to *find* patterns in text. Think of grep as your search-and-locate tool.

### Basic Pattern Matching

Find all error lines:

```bash
grep "ERROR" access.log
```

Find lines containing a specific status code:

```bash
grep "500" access.log  # Find all internal server errors
```

### Counting Occurrences with `-c`

How many errors do we have?

```bash
grep -c "ERROR" access.log
# Output: 3
```

How many 429 (rate limit) warnings?

```bash
grep -c "429" access.log
# Output: 2
```

### Showing Context with `-B` and `-A`

See lines *before* an error (to see what happened just before):

```bash
grep -B 1 "ERROR" access.log  # -B = before, show 1 line before
```

See lines *after* an error:

```bash
grep -A 2 "ERROR" access.log   # -A = after, show 2 lines after
```

See lines on both sides:

```bash
grep -C 2 "500" access.log     # -C = context, 2 lines before and after
```

### Regex Patterns

Find all 5xx errors (500, 503, etc.):

```bash
grep "5[0-9][0-9]" access.log  # Regex: 5 followed by two digits
```

Find requests from a specific IP range:

```bash
grep "192\.168\.1\." access.log  # Escape dots to match literal periods
```

## Section 2: sed — The Surgeon

**When to use it:** You need to *modify* or *transform* text. Think of sed as your find-and-replace tool for streams.

### Find and Replace

Replace all "WARN" with "WARNING" for clarity:

```bash
sed 's/WARN/WARNING/g' access.log
```

### Masking Sensitive Data

Before sharing logs with a third party, mask IP addresses:

```bash
sed 's/[0-9]\{1,3\}\.[0-9]\{1,3\}\.[0-9]\{1,3\}\.[0-9]\{1,3\}/xxx.xxx.xxx.xxx/g' access.log
```

Replace API keys or tokens:

```bash
sed 's/api_key=[A-Za-z0-9]*/api_key=REDACTED/g' access.log
```

### In-Place Editing

Edit the file directly (no output to stdout):

```bash
sed -i 's/INFO/INFORMATION/g' access.log  # -i = in-place, changes the file
```

### Extract and Transform

Add a prefix to each line:

```bash
sed 's/^/LOG: /' access.log  # ^ = start of line
```

Remove the timestamp column:

```bash
sed 's/^[0-9-]* [0-9:]* //' access.log  # Strip timestamp and space
```

## Section 3: awk — The Analyst

**When to use it:** You need to *parse columns* and *calculate metrics*. Think of awk as your mini-reporting engine.

### Field Extraction

By default, awk splits on whitespace. Let's extract just the response times (column 7):

```bash
awk '{ print $7 }' access.log
# Output:
# 45ms
# 38ms
# 1203ms
# ...

# Remove the "ms" suffix for calculations:
awk '{ print substr($7, 1, length($7)-2) }' access.log
```

### Arithmetic Operations

Calculate the average response time:

```bash
awk '{
  # Remove "ms" suffix and convert to number
  gsub(/ms/, "", $7)
  sum += $7
  count++
}
END { print "Average response time: " sum/count "ms" }' access.log
```

### Conditional Filtering

Print only lines where response time exceeds 500ms:

```bash
awk '{
  gsub(/ms/, "", $7)  # Remove "ms"
  if ($7 > 500) print $0  # $0 = entire line
}' access.log
```

### Column-Based Filtering

Print only the IP and endpoint from each line:

```bash
awk '{ print $4, $6 }' access.log
# Output:
# 192.168.1.105 /api/v2/users
# 10.0.3.87 /api/v2/products
# ...
```

### Pattern Matching Within Columns

Find all lines with ERROR and print IP and endpoint:

```bash
awk '/ERROR/ { print $4, $6 }' access.log
```

## The Grand Finale: The Pipeline

Now let's combine all three tools to solve a real problem:

**"Find all slow errors (5xx status, response time > 500ms), mask the IPs, and calculate the average response time."**

```bash
grep "5[0-9][0-9]" access.log |           # 1. Filter 5xx errors
  sed 's/[0-9]\{1,3\}\.[0-9]\{1,3\}\.[0-9]\{1,3\}\.[0-9]\{1,3\}/xxx.xxx.xxx.xxx/g' |  # 2. Mask IPs
    awk '{
      gsub(/ms/, "", $7)  # Remove "ms"
      sum += $7
      count++
      print $1, $2, $3, "xxx.xxx.xxx.xxx", $5, $6, $7"ms"  # Reformat line
    }
    END {
      print "---"
      print "Total slow errors: " count
      print "Average response time: " sum/count "ms"
    }'
```

Output:

```bash
2026-05-23 02:14:36 ERROR xxx.xxx.xxx.xxx GET /api/v2/inventory 500 892ms
2026-05-23 02:14:38 ERROR xxx.xxx.xxx.xxx POST /api/v2/checkout 500 2341ms
2026-05-23 02:14:41 ERROR xxx.xxx.xxx.xxx GET /api/v2/products 503 567ms
---
Total slow errors: 3
Average response time: 1266.67ms
```

### Another Powerful Example

**"Which endpoint has the most errors?"**

```bash
grep "ERROR" access.log | awk '{ print $6 }' | sort | uniq -c | sort -rn
```

## Cheat Sheet Summary

| Task | Command | Notes |
|------|---------|-------|
| Find pattern | `grep "ERROR" file.log` | Basic search |
| Count matches | `grep -c "500" file.log` | Returns number only |
| Context lines | `grep -C 2 "ERROR" file.log` | Show 2 lines before & after |
| Case-insensitive | `grep -i "error" file.log` | Ignore case |
| Invert match | `grep -v "INFO" file.log` | Show lines NOT matching |
| Find and replace | `sed 's/old/new/g' file` | Global replacement |
| Mask IPs | `sed 's/[0-9.]*/REDACTED/g'` | Hide sensitive data |
| Edit in place | `sed -i 's/old/new/g' file` | Changes file directly |
| Extract column | `awk '{ print $4 }' file` | Column number |
| Sum values | `awk '{ sum+=$7 } END { print sum }'` | Calculate totals |
| Filter by condition | `awk '$7 > 500' file` | Rows where column 7 > 500 |
| Pattern in column | `awk '/ERROR/ { print $6 }'` | Column matching regex |
